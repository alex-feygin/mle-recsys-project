# Подготовка виртуальной машины

## Склонируйте репозиторий

Склонируйте репозиторий проекта:

```
git clone https://github.com/yandex-praktikum/mle-project-sprint-4-v001.git
```

## Активируйте виртуальное окружение

Используйте то же самое виртуальное окружение, что и созданное для работы с уроками. Если его не существует, то его следует создать.

Создать новое виртуальное окружение можно командой:

```
python3 -m venv env_recsys_start
```

После его инициализации следующей командой

```
. env_recsys_start/bin/activate
```

установите в него необходимые Python-пакеты следующей командой

```
pip install -r requirements.txt
```

# Сервис рекомендаций

Сервис рекомендаций реализован в виде трёх отдельных FastAPI-микросервисов, взаимодействующих по HTTP.

## Архитектура


| Компонент               | URL                     | Роль                                           |
| ----------------------- | ----------------------- | ---------------------------------------------- |
| `EventStore`            | `http://127.0.0.1:8020` | Хранение истории событий пользователя в памяти |
| `RecommendationService` | `http://127.0.0.1:8010` | Офлайн-, онлайн- и смешанные рекомендации      |
| Шлюз (Gateway)          | `http://127.0.0.1:8000` | Оркестрирует оба подсервиса, отдаёт единый API |


## Структура проекта

```
├── recommendation_service.py   # Шлюз (порт 8000) — компонует два подсервиса
├── recommendations_app.py      # RecommendationService (порт 8010) — офлайн/онлайн/blend
├── events_app.py               # EventStore (порт 8020) — запись и чтение событий
├── test_service.py             # Интеграционные тесты (3 сценария)
├── conftest.py                 # pytest-плагин: запись результатов в test_service.log
└── recsys/
    ├── recommendations/
    │   ├── personal_als.parquet  # Персональные рекомендации ALS (user_id, item_id, score, rank)
    │   ├── cold_recs.parquet     # Популярные треки для холодного старта (item_id, score)
    │   └── similar.parquet       # Косинусная близость треков (track_id, similar_track_id, score, rank)
    └── data/
        └── items.parquet         # Метаданные треков
```

## Запуск сервиса

Запустите все три сервиса в отдельных терминалах:

```bash
uvicorn events_app:app             --host 0.0.0.0 --port 8020
uvicorn recommendations_app:app    --host 0.0.0.0 --port 8010
uvicorn recommendation_service:app --host 0.0.0.0 --port 8000
```

Документация Swagger доступна по адресам:

- Шлюз:                  [http://localhost:8000/docs](http://localhost:8000/docs)
- RecommendationService: [http://localhost:8010/docs](http://localhost:8010/docs)
- EventStore:            [http://localhost:8020/docs](http://localhost:8020/docs)

## Эндпоинты шлюза


| Метод  | Эндпоинт                               | Описание                                                         |
| ------ | -------------------------------------- | ---------------------------------------------------------------- |
| `POST` | `/put_event?user_id=&item_id=`         | Сохранить событие взаимодействия пользователя                    |
| `GET`  | `/get_events?user_id=&k=`              | Получить последние события пользователя                          |
| `GET`  | `/recommendations_offline?user_id=&k=` | Офлайн-рекомендации ALS (холодный старт для новых пользователей) |
| `GET`  | `/recommendations_online?user_id=&k=`  | Онлайн-рекомендации на основе недавних событий                   |
| `GET`  | `/recommendations?user_id=&k=`         | Смешанные офлайн + онлайн рекомендации                           |


Все эндпоинты рекомендаций возвращают `{"recs": [...]}`.

### Логика онлайн-рекомендаций

1. Взять последние `ONLINE_HISTORY_DEPTH=3` события пользователя
2. Для каждого трека-источника найти похожие треки в `similar.parquet`
3. Объединить всех кандидатов
4. Отсортировать по убыванию оценки схожести
5. Удалить дубликаты (оставить первое вхождение)
6. Вернуть топ-`k`

### Стратегия смешивания (blend)

1. Получить офлайн-рекомендации (персональные ALS или холодный старт)
2. Получить онлайн-рекомендации (контентные, по последним 3 событиям)
3. Чередовать элементы: offline[0], online[0], offline[1], online[1], ...
4. Дописать оставшийся хвост
5. Удалить дубликаты (оставить первое вхождение)
6. Вернуть первые `k` элементов

# Инструкции для тестирования сервиса

Код для тестирования находится в файле `test_service.py`. Перед запуском убедитесь, что все три сервиса запущены и полностью загружены (дождитесь появления сообщения `Application startup complete` в каждом терминале).

```bash
pytest test_service.py -v
```

Результаты тестов записываются в файл `test_service.log`.

## Тестовые сценарии

- **Сценарий 1** — Холодный пользователь (нет персональных рекомендаций, нет истории): офлайн → популярные треки; онлайн → пусто; смешанные → популярные треки
- **Сценарий 2** — Тёплый пользователь без истории событий: офлайн → персональные ALS-рекомендации; онлайн → пусто; смешанные → персональные ALS-рекомендации
- **Сценарий 3** — Тёплый пользователь с историей событий: онлайн → контентные рекомендации; смешанные → чередование офлайн и онлайн, без дубликатов, длина = k
